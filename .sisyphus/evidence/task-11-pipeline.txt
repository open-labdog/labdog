Key design decisions:
- run_sync_playbook requires (job_id, host_id) → SyncJob created per host before dispatch
- Skips hosts with already-running syncs (mirrors api/sync.py pattern)
- SSH key resolved from SSHKey model for SSH auth repos
- triggered_by_user_id=None distinguishes gitops-triggered from user-triggered syncs
- Idempotency: same commit SHA processed twice → second is no-op
- Missing YAML file → group status "error", other groups still processed
- Import failure → no sync triggered for that group
- Temp repo directory cleaned in finally block
